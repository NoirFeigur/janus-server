## 里程碑总览

| 里程碑 | 主题 | 状态 | 一句话范围 |
|---|---|---|---|
| **M0** | 项目基座 | ✅ 完成 | 可启动 + 连库 + 16 表迁移 + 统一信封/异常 + i18n 骨架 + 结构化日志 + Redis 客户端 + 真实就绪探针 |
| **M1** | 账户 / 凭证 / RBAC | ⬜ 待开始 | 本地账户体系 + 密码哈希 + sk-key 校验 + 鉴权依赖 + 角色/菜单/权限判定 + 数据范围 |
| **M2** | LLM 网关（热路径） | ⬜ 待开始 | 鉴权后的 LLM 请求流 + 模型路由 + 上游调用 + 配额扣减 + 用量记账 |
| **M3** | 管理后台 API | ⬜ 待开始 | 受保护的 CRUD（账户/角色/菜单/模型目录/配额/分配/用量） + 分页 |
| **M4** | MCP 服务器 | ⬜ 待开始 | 经 FastMCP 暴露服务层能力（零新表，复用账户/key 鉴权，不绕过 RBAC） |
| **M5** | ARQ 后台任务 | ⬜ 待开始 | 用量汇总 / 配额重置 / 账单滚动 / 清理（从请求热路径剥离） |
| **M6** | 企微对接 | ⬜ 待开始 | 企微 OAuth / JWT 验签作为「第二凭证入口」（账户核心稳固后再接） |

**排序铁律**（来自架构评审，违反会导致返工）：

- **M1 先于一切业务面**：网关与管理后台都依赖鉴权依赖与 sk-key 解析；先做地基，否则各自重造一套鉴权。
- **M2 配额原语就地建**：Redis 发布订阅 / 配额计数原语随 M2 的消费者落地，**不在 M0 提前堆砌**（无消费者的基建是过度工程）。
- **M3 依赖 M1 的 RBAC**：管理后台 CRUD 的权限门禁必须建立在 M1 的角色/权限判定之上，否则权限是后补的、有漏洞。
- **M4 晚于 M2/M3**：MCP 经服务层暴露能力，必须等服务边界稳定，否则会重复业务逻辑。
- **M6 最后做**：企微是**外部适配器**，不是地基。账户/RBAC 核心用平台自建凭证（密码 + sk-key）即可独立闭环且完全可测；企微 OAuth 作为额外的凭证入口后挂。

## M0：项目基座 ✅

**目标**：项目可启动、连开发库、迁移可跑、横切基础设施就位。**已全部完成。**

### 已落地

**工程骨架**
- `pyproject.toml`：hatchling 构建；运行时依赖（fastapi/sqlalchemy[asyncio]/asyncpg/pydantic/pydantic-settings/
  alembic/arq/redis/httpx/litellm/mcp/pyjwt[crypto]/cryptography/uvicorn + **structlog** + **argon2-cffi**）；
  dev 依赖（pytest/pytest-asyncio/aiosqlite/mypy/ruff + **pytest-cov** + **fakeredis**）；全部带版本上界。
- `ruff`（line-length 100，select E/F/I/B/UP/SIM）+ `mypy --strict`（pydantic plugin）配置就位。
- 三道质量门可跑：`uv run ruff check` / `uv run mypy src` / `uv run pytest`。

**数据层**
- `src/db/base.py`：`DeclarativeBase` + 三类实体基类（BaseEntity / LogEntity / LinkEntity，雪花主键、软删、审计列）。
- `src/db/session.py`：async engine + `async_session_factory` + `get_session()` 依赖。
- `src/db/repository.py`：`BaseRepository[Model]` 泛型 CRUD。
- `src/db/models/`：16 张 ORM 模型（identity / credential / model_catalog / grant / usage / quota）。
- `src/core/snowflake.py`：雪花 ID 生成器（开发环境 worker-id 固定 0）。
- `alembic/`：首迁移 `initial_schema_16_tables`，`alembic upgrade head` 可跑。

**横切基础设施**
- `src/config.py`：pydantic-settings（DB/Redis URL、JWT 公钥/JWKS、Fernet key、locale、**log_level/log_json**）。
- `src/enums.py`：集中 StrEnum（含 `ErrorCode`），每个成员强制行内注释。
- `src/responses.py`：统一响应信封（方案 B，`success` 判别；成功体仅 `data`，错误体仅 `code`）。
- `src/exceptions.py`：`AppError` 根异常 + 4 个 RFC 9457 `problem+json` 拦截器；i18n 纯 code 路线。
- `src/core/i18n/`：locale `ContextVar`（`get_locale`/`set_locale`/`normalize_locale`） + 解析中间件
  （query → cookie → Accept-Language → 默认）。
- `src/core/logging.py`：**structlog 结构化日志**——`configure_logging()`（进程级幂等）+ `get_logger()` +
  `bind_trace_id()` / `clear_context()`；用 `merge_contextvars` 让 trace_id 随请求协程自动流入每行日志；
  `log_json` 切换 JSON 行（生产）/ 彩色控制台（开发）。
- `src/core/redis.py`：**最小异步 Redis 客户端**——`get_redis()`（进程级单例，连接池）/ `close_redis()` / `ping()`。
  仅生命周期 + 健康探测；发布订阅、配额计数原语**不在这里**（随 M2 消费者落地）。
- `src/main.py`：app 工厂 + **lifespan**（启动初始化日志、关闭回收 Redis）+ `TraceIdMiddleware`
  （绑定/清理 trace_id）+ `LocaleMiddleware` + 异常拦截器；`/health/live`（裸格式存活）+
  **`/health/ready`（真实探测 PG + Redis，任一不可达返回 503）**。

**测试**
- `tests/conftest.py`：内存 SQLite engine/session 工厂 + **fakeredis 异步 fixture**（替换模块单例，不碰共享 Redis）。
- 43 个测试全绿：snowflake / repository / base / responses / exceptions / **logging / redis / health**。
- 单测全部用内存 SQLite + fakeredis，零触碰共享 PG/Redis 实例。

### M0 遗留（不阻塞，记录防遗忘）
- Pydantic 422 校验错误的 `errors[].msg` 当前发 Pydantic 英文原文（i18n 方向 A/B 待定，M1 处理）。

## M1：账户 / 凭证 / RBAC ⬜

**目标**：平台自建的账户体系 + 鉴权地基。**零外部依赖、完全可测**（不碰企微）。

### 范围
- `src/core/security.py`：
  - 密码哈希（**argon2-cffi**，`PasswordHasher`，OWASP 首选）。
  - sk-key 校验（`sk-xxxx` → sha256 哈希 → DB 查 `api_key.key_hash`）。
  - 平台自签 JWT（平台作为签发方，本地登录换发；**不含企微验签**）。
- `src/auth/`（领域包）：
  - `repository.py`：用户 / 角色 / key 查询（继承 BaseRepository）。
  - `service.py`：本地登录校验 / sk-key 解析 / RBAC 权限判定（编排 repository）。
  - `dependencies.py`：`CurrentUser` / `RequiredPerms` / `AuthenticatedApiKey`。
  - `router.py`：登录 / token 校验 / 当前用户。
  - `schemas.py` / `constants.py` / `exceptions.py`。
- RBAC 判定：基于已有的 `sys_role` / `sys_menu` / `sys_user_role` / `sys_role_menu` 表 +
  6 级 `data_scope`（数据范围）原语。

### 边界（未决，开工前确认）
- M1 是「仅鉴权读侧 + 认证」（用户/角色/部门 CRUD 留到 M3 admin），还是含基础管理 CRUD？
  —— **此问题用户此前未答**，M1 开工前需明确。

### 依赖
- 依赖 M0 的：`core/security.py` 落点、`get_session()`、`BaseRepository`、`enums.ErrorCode`、统一信封。

## M2：LLM 网关（热路径）⬜

**目标**：鉴权后的 LLM 请求全流程。系统的核心价值面。

### 范围
- `src/gateway/router.py` + `providers/`：客户端协议原生入口（OpenAI `/v1/chat/completions`、
  Anthropic `/v1/messages`、Google `generateContent`）。
- `src/gateway/router_factory.py`：`litellm.Router` 装配（号池 → deployment）+ 韧性配置（fallback）。
- `src/gateway/service.py`：路由 / fallback / 配额扣减 / 记账落 `usage_record`。
- `src/gateway/dependencies.py`：`RateLimitGated` / `QuotaGated`。
- **Redis 配额/限流原语就地建**（计数器 + 发布订阅失效）——这是它们的第一个消费者。
- 新依赖随消费者引入：`tenacity`（上游重试）、`respx`（mock 上游 httpx 测协议路由/降级）。

### 依赖
- 依赖 M1 的：`AuthenticatedApiKey` 依赖、sk-key 解析（否则网关会重造鉴权）。

## M3：管理后台 API ⬜

**目标**：受 RBAC 保护的资源 CRUD。

### 范围
- `src/admin/` 资源子包：`users/`（用户/角色/部门）、`credentials/`（api_key 管理）、
  `catalog/`（逻辑模型/渠道/号池）、`grants/`（模型分配）、`usage/`（用量/内部账单读侧）、`quota/`（配额配置）。
- `src/core/pagination.py`：**通用分页 schema**——随第一个列表端点落地（不提前建）。
- 敏感字段加解密落各自 service：`channel_key.api_key_encrypted`（Fernet）、`sys_user.password`（argon2，复用 M1）。

### 依赖
- 依赖 M1 的 RBAC 判定 + `RequiredPerms` 依赖（权限门禁建立在已有判定之上）。

## M4：MCP 服务器 ⬜

**目标**：平台自身即标准 MCP server，经 FastMCP 暴露服务层能力。

### 范围
- `src/mcp/server.py`：FastMCP 实例（`stateless_http=True`）+ `TokenVerifier`（复用账户/key 鉴权）。
- `src/mcp/tools/`：MCP 工具实现（平台内代码，内部调第三方 REST）。
- **零新表**；MCP 调用复用 M1 鉴权，**不绕过 RBAC / 配额规则**。

### 依赖
- 依赖 M2/M3 的服务边界（否则会重复业务逻辑）。

## M5：ARQ 后台任务 ⬜

**目标**：把非热路径的重活/周期活从请求流剥离。

### 范围
- `src/tasks/`：`worker.py`（WorkerSettings + 进程入口）、`schedule.py`（cron 调度表）、`registry.py`（任务注册）。
- 领域 `tasks.py`：夜间用量汇总（gateway/admin）、周期配额重置（quota）、账单滚动、清理。
- **scheduler 单实例铁律**；任务幂等（可安全重跑）。

### 依赖
- 依赖 M2/M3 的用量与配额数据模型已稳定。

## M6：企微对接 ⬜

**目标**：企微 OAuth / JWT 验签作为「第二凭证入口」。

### 范围
- 企微 JWT 验签（无状态，公钥/JWKS 校验）。
- 企微 OAuth 登录流 → 命中 `sys_user_oauth` 硬匹配 → 解析到平台账户。
- 接入 `core/security.py` 的 JWT 验签分支（与 M1 的平台自签 JWT 并存）。

### 前置（开工前确认）
- 企微 JWT claims 契约 —— **用户此前定「后面再说」**，M6 开工前需明确签发方与 claim 结构。

### 依赖
- 依赖 M1 的账户核心（企微只是额外凭证入口，账户/RBAC 已独立闭环）。

## 全局约束（贯穿所有里程碑）

- **类型安全零容忍**：禁 `# type: ignore`，`mypy --strict` 是 CI 门禁。
- **测试是收工前置**：新功能/修 bug 必带测试（红 → 绿）；禁删/skip 失败用例骗 CI；单测不碰共享 PG/Redis。
- **四层单向依赖**：router → service → repository → db；router 薄、service 厚。
- **窄腰 + 最小依赖**：新依赖必带版本上界；优先 stdlib > 焦点库 > 套件（拒绝 kitchen-sink util）。
- **prompt 缓存神圣**（网关侧）：不中途改上下文/换工具集/重建系统提示。
- **i18n 纯 code**：后端 locale-agnostic 只发 code，前端拥有展示文本；枚举 label 例外（后端单源 + codegen 同步）；
  不建字典表。
- **配置/密钥分离**：`.env` 仅放密钥；行为配置走 `config.py`。
