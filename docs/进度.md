## 里程碑总览

| 里程碑 | 主题 | 状态 | 一句话范围 |
|---|---|---|---|
| **M0** | 项目基座 | ✅ 完成 | 可启动 + 连库 + 16 表迁移 + 统一信封/异常 + i18n 骨架 + 结构化日志 + Redis 客户端 + 真实就绪探针 |
| **M1** | 账户 / 凭证 / RBAC | ✅ 完成 | 本地账密登录 + argon2 哈希 + sk-key 校验 + 平台自签 RS256 JWT + 鉴权依赖 + RBAC 权限判定 + 数据范围 + 用户/角色/部门管理 CRUD |
| **M2** | LLM 网关（热路径） | ⬜ 待开始 | 鉴权后的 LLM 请求流 + 模型路由 + 上游调用 + 配额扣减 + 用量记账 |
| **M3** | 管理后台 API | 🟡 部分完成 | 用户/角色/部门 CRUD 已在 M1 提前落地；剩余 credentials/catalog/grants/usage/quota + 分页待做 |
| **M4** | MCP 服务器 | ⬜ 待开始 | 经 FastMCP 暴露服务层能力（零新表，复用账户/key 鉴权，不绕过 RBAC） |
| **M5** | ARQ 后台任务 | ⬜ 待开始 | 用量汇总 / 配额重置 / 账单滚动 / 清理（从请求热路径剥离） |
| **M6** | 企微对接 | ⬜ 待开始 | 企微 OAuth / JWT 验签作为「第二凭证入口」（账户核心稳固后再接） |

**排序铁律**（来自架构评审，违反会导致返工）：

- **M1 先于一切业务面**：网关与管理后台都依赖鉴权依赖与 sk-key 解析；先做地基，否则各自重造一套鉴权。
- **M2 配额原语就地建**：Redis 发布订阅 / 配额计数原语随 M2 的消费者落地，**不在 M0 提前堆砌**（无消费者的基建是过度工程）。
- **M3 依赖 M1 的 RBAC**：管理后台 CRUD 的权限门禁必须建立在 M1 的角色/权限判定之上，否则权限是后补的、有漏洞。
- **M4 晚于 M2/M3**：MCP 经服务层暴露能力，必须等服务边界稳定，否则会重复业务逻辑。
- **M6 最后做**：企微是**外部适配器**，不是地基。账户/RBAC 核心用平台自建凭证（密码 + sk-key）即可独立闭环且完全可测；企微 OAuth 作为额外的凭证入口后挂。

## M1：账户 / 凭证 / RBAC ✅

**目标**：平台自建的账户体系 + 鉴权地基。**零外部依赖、完全可测**（不碰企微）。**已全部完成。**

### 已落地

**核心安全层** `src/core/security.py`
- 密码哈希（**argon2-cffi** `PasswordHasher`，OWASP 首选；verify 不匹配/畸形哈希均折叠为 False）。
- sk-key 校验（`sk-xxxx` → sha256 → DB 查 `api_key.key_hash`）。
- 平台自签 JWT（平台作为签发方，本地登录换发；**不含企微验签**）。算法**硬锁 RS256**——
  encode/decode 都用常量，decode 显式传 `algorithms=["RS256"]`，绝不信任 token header 自报的 alg
  （防算法混淆攻击）。`PlatformAccessClaims` 设 `extra="forbid"` 拒绝多余 claim 走私。
  claims 最小集 `sub/iat/exp/token_use`；权限不入 JWT，每请求从 DB 取。

**auth 领域包** `src/auth/`
- `router.py`：`POST /auth/login`（账密换 access token）、`GET /auth/me`（当前账户）。
- `repository.py`：`AuthRepository`——用户/key 查询 + 聚合 `User→UserRole→Role→RoleMenu→Menu` 出权限码集合。
- `service.py`：`AuthService`——账密校验、token/api-key 解析、RBAC 判定、数据范围解析；
  frozen DTO `AuthenticatedAccount` + `DataScopeFilter`；super-admin `*:*:*` 短路为 unrestricted；
  `data_scope` 邻接表遍历 `dept_and_child` 子树。
- `dependencies.py`：`CurrentAccount` / `TraceId` / `RequiredPerms`（凭据解析 X-API-Key 优先→Bearer）。
- `schemas.py`：登录/token/当前用户（雪花 int64 ID 在 wire 序列化为 string，防 JS Number 精度丢失）。
- 认证失败折叠为不透明码：坏凭据/坏 token/禁用账户 → `auth.invalid_token` 401，越权 → `auth.forbidden` 403。

**用户/角色/部门管理 CRUD** `src/admin/`（提前完成 M3 的一部分）
- `departments/`：CRUD + parent 校验 + reparent 环路守卫 + 删除时子部门/成员阻断。
- `roles/`：CRUD + RoleMenu(权限)/RoleDept(custom 范围) 链管理 + code 唯一 + 删除时物理清 UserRole + custom dept 校验。
- `users/`：CRUD + UserRole 分配 + argon2 哈希（永不出 read model）+ username/employee_no 唯一。
  **3 道安全守卫**：(1) 越权创建/移动防护（`_require_department_in_scope`）、
  (2) 角色分配提权防护（`_require_assignable_roles`，被授权限须为 actor 权限子集）、(3) 工号唯一。
- 挂 `/admin`，perm 门禁 `system:{dept,role,user}:*`；`data_scope` 仅 list + mutation 应用。

**RBAC 设计决策**
- 权限匹配定为**精确匹配 + 单一 `*:*:*` super-admin 哨兵**（否决段级通配；分组授权由菜单父节点表达）。
- M1 仅 access token；refresh token 留 M6。

### 质量门
- pytest **112 passed**、mypy --strict **56 files 0 error**、ruff clean。
- **Oracle 二审 BLOCK LIFTED**，2 CRITICAL + 4 IMPORTANT + MINOR 全部 VERIFIED-CORRECT。

### 依赖
- 依赖 M0 的：`core/security.py` 落点、`get_session()`、`BaseRepository`、`enums.ErrorCode`、统一信封。

## 全局约束（贯穿所有里程碑）

- **类型安全零容忍**：禁 `# type: ignore`，`mypy --strict` 是 CI 门禁。
- **测试是收工前置**：新功能/修 bug 必带测试（红 → 绿）；禁删/skip 失败用例骗 CI；单测不碰共享 PG/Redis。
- **四层单向依赖**：router → service → repository → db；router 薄、service 厚。
- **窄腰 + 最小依赖**：新依赖必带版本上界；优先 stdlib > 焦点库 > 套件（拒绝 kitchen-sink util）。
- **prompt 缓存神圣**（网关侧）：不中途改上下文/换工具集/重建系统提示。
- **i18n 纯 code**：后端 locale-agnostic 只发 code，前端拥有展示文本；枚举 label 例外（后端单源 + codegen 同步）；
  不建字典表。
- **配置/密钥分离**：`.env` 仅放密钥；行为配置走 `config.py`。
