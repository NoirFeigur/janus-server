## M0 已完成明细

### 工具链与依赖
- `uv` 装入 `.venv`；`uv sync --extra dev` 生成 `uv.lock` 并装齐依赖。
- 依赖全部带上界（供应链安全，见 README「依赖锁定」）。

### 配置（`src/config.py` + `.env`）
- 新增 `redis_arq_url`：ARQ 队列走 Redis db 1，与业务缓存/配额计数（db 0）物理隔离。
- 连接串设计保持「完整 URL 串」（`database_url` / `redis_url` / `redis_arq_url`），不拆离散 host/port 字段——SQLAlchemy / redis 原生消费 URL。
- 密钥仅入 `.env`（已 gitignore），`.env.example` 为模板（不含真实值）。

### 持久层（`src/db/` + `src/core/snowflake.py`）
- `core/snowflake.py`：线程安全雪花 ID 生成器（41 位毫秒 + 10 位 worker-id + 12 位序列），时钟回拨自旋等待，开发环境 worker-id 默认 0（生产 Redis 租约 fail-fast 为运行时增量，后做）。
- `db/base.py`：单一 `DeclarativeBase` + **三类实体基类**：
  - `BaseEntity`——配置/业务表（雪花主键 + 软删 `is_deleted` + 审计列）；
  - `LogEntity`——append-only 流水（仅 id + created_at）；
  - `LinkEntity`——多对多关联表（id + created_at，物理删，无审计列）。
  - 时间列统一 `timestamptz`（UTC，DB 端 `func.now()` 生成）。
- `db/session.py`：async engine + `async_sessionmaker` + `get_session()` 依赖；URL 取自 Settings（`.env`），非硬编码。
- `db/repository.py`：`BaseRepository[Model]` 泛型 CRUD（`get` / `list` / `create` / `update` / `soft_delete`），软删表默认排除 `is_deleted=true`，接收注入的 `AsyncSession`（不自建 engine）。

### ORM 模型（`src/db/models/`，16 张表）
按数据模型 v0.5.4 全部落地，枚举成员强制行内业务语义注释：

| 文件 | 表 |
|---|---|
| `identity.py` | `sys_user` / `sys_department` / `sys_user_oauth` / `sys_role` / `sys_menu` / `sys_user_role` / `sys_role_menu` / `sys_role_dept` |
| `credential.py` | `api_key` |
| `model_catalog.py` | `upstream_channel` / `channel_key` / `logical_model` / `model_deployment` |
| `grant.py` | `user_model_grant` |
| `usage.py` | `usage_record` |
| `quota.py` | `quota` |

- 全平台无物理外键（§0.7）：关联列存目标雪花 id + 普通索引，引用完整性由 service 层维护。
- 软删表唯一约束走「部分唯一索引」（`postgresql_where="is_deleted = false"`）。
- `quota` 唯一索引用 `NULLS NOT DISTINCT`（PG15+）+ `CheckConstraint`（global ⇒ scope_id 为空；非 global ⇒ scope_id 必填）。

### 迁移（`alembic/`）
- `env.py` 改为 async-aware，URL 取自 Settings（覆盖 `alembic.ini` 占位），`target_metadata = Base.metadata` 并 import 全部模型。
- 首迁移 autogenerate 正确捕获：16 张表、`NULLS NOT DISTINCT`、CHECK 约束、13 个部分唯一索引。
- `alembic upgrade head` 在开发库 `postgres` 建表成功（实测 PostgreSQL 16.14）。

### 启动自测
- `uvicorn src.main:app` 启动无告警；`/health/live`、`/health/ready` 均 200；OpenAPI 正常。

## 已知遗留 / 待办

- **`exceptions.py` 2 处 mypy 报错**（脚手架既有，非本轮代码）：FastAPI `add_exception_handler` 形参型变模式，仅类型层、无运行时影响。后续小修。
- **`/health/ready` 当前为硬编码 `ok`**：M1 起应真实探活 PG + Redis（连接 ping）。
- **雪花 worker-id 生产租约**：当前开发环境固定 0；多副本上线前需补 Redis SPOP 租约 + 心跳续租 + fail-fast（§0.2）。
- **敏感字段加解密**：`channel_key.api_key_encrypted`（Fernet 可逆）、`api_key.key_hash`（sha256）、`sys_user.password`（argon2/bcrypt）落地在各自 service，M1/M3 实现。

## 如何运行

```bash
# 装依赖（首次）
uv sync --extra dev

# 跑迁移（开发库）
uv run alembic upgrade head

# 启动
uv run uvicorn src.main:app --reload --port 8000

# 质量门（提交前必过，见 README「单元测试规范」）
uv run pytest
uv run ruff check
uv run mypy src
```
